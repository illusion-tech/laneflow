## 2. 已核验契约

- ADR 0005 与 Accepted `core-id-handles.md` 定义 external ID、typed handle、generation 和稳定 update order。
- Accepted `lane-graph.md`、`route-system.md` 与 `data-format.md` 对 edge connection、route edge sequence、handle 归一化和 validation 分层的描述一致。
- 收口时的 [v0.2 schema](https://github.com/illusion-tech/laneflow/blob/cbc573e673fa5e1cc61879e1f4a3e84e6b6afec4/schemas/laneflow-data-v0.2.schema.json) 固定 `formatVersion: "0.2"`、`units.distance: "meter"`、external ID 形态和 closed JSON shape；该 immutable permalink 是历史证据，不是当前 active contract。
- example route data 可以通过 JSON Schema Draft 2020-12 校验、domain validation，并驱动 Core route following 完成。
- Core hot path 使用 handle 和稳定 update order，不在每 tick 克隆 external ID 或按字符串重新排序。
- v0.1 `core-runtime.md` 已明确为历史 Review 基线；v0.2 Accepted 文档和 ADR 的现行契约优先。

## 5. 性能观察

同机本轮观测：

- steady 10k vehicles / 60 steps：median 约 `8.30 ms`。
- transition-heavy 10k vehicles / 60 steps：median 约 `10.69 ms`。

与 #39 记录相比，steady 约慢 17%，transition-heavy 约快 4.8%；两者均显著低于 `1 s / 60 steps` 软门槛。Criterion 本地缓存基线的来源不足以单独形成回归结论，因此该差异不阻断 v0.2；只有在同环境受控复测中可复现时，才应创建独立性能 Issue。

## 6. 后续版本政策

v0.3 起，LaneFlow 依据 ADR 0008 在 1.0 前只维护单一 active data format。v0.2 schema、fixture 和 production loader 路径已从当前树移除，但本报告、immutable commit 和对应 GitHub Gate 证据继续保存其收口事实。该变化不追溯修改本报告记录的测试结论，也不表示当前 loader 承诺兼容 v0.2。

## 7. 结论

截至本次审阅，未发现阻断 v0.2 Lane Graph + Route 收口的 Core 正确性、数据契约、测试、依赖许可或治理缺口。后续 milestone 可以把本基线作为稳定输入；新增行为仍须通过各自 G0-G4，不得把本结论扩展为未审阅能力的默认批准。
