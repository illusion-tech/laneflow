## 3. 设计与架构契约

- ADR 0009 保持 indication、spatial Gate/StopLine、jurisdiction policy 与不可绕过 Core safety 的分层；SignalController 不硬编码国家或转向规则。
- StopLine v0.4 固定在 edge end；MovementGate 由 directed `(fromEdge, toEdge)` 标识，并显式绑定 `group` 或 `none`。
- fixed-time state 只由 world absolute integer time、canonical offset 和 ordered half-open phases 推导；`u128` 中间计算避免 near-`u64::MAX` 加法溢出。
- tick-start committed snapshot 决定本 tick vehicle permission；candidate signal state 与 vehicle state 成功后一起原子提交。
- green 表示 signal-layer protected allow，yellow/red 表示 restrictive deny-and-stop；`none` 只表示 Signals layer 不约束，不表达未来法规或冲突让行。
- vehicle events 按 stable update order 产生，之后 controller normalization order 产生 phase/group events；query snapshot 是当前 authority，events 只表示 transition。
- Adapter 仍只允许 query/render，不拥有交通规则；v0.4 没有新增或冻结 Adapter ABI。

Accepted `signal-system.md`、ADR 0009、`data-format.md`、`data-loading.md`、architecture 和当前代码在这些边界上一致。

## 4. 实现与不变量审阅

- `SignalRegistry` 在 initialization 完成 external ID、引用、ownership、complete phase state、Gate coverage、StopLine 与 route-final 约束 normalization；tick hot path 只使用 handles/dense indices。
- `SignalRuntimeState` 保存 compact controller/group snapshots；private `has_restrictive_group` bit 允许全 green 或无 controlled group 时跳过逐车 SignalStop denial 搜索。
- route registration 预解析 transition Gate 与 nearest-controlled metadata；每车不扫描全部 Gates/Controllers/Groups，traversal 只处理实际 route occurrence。
- denied Gate 形成 route-distance SignalStop constraint；纵向 pipeline 保持 IIDM、safe-speed、SignalStop、no-overlap projection 与 route transition 的确定顺序。
- red/yellow exact boundary、nearest denied Gate、多 Gate、repeated physical edge、shared StopLine、queue/release 与 dual projection 都有定向测试。
- failed tick、non-finite signal calculation、time overflow 和 invalid input 不改变 committed vehicle/signal/tick authority state。
- arbitrary-time resolver 只存在于 private test path；public API 没有为测试暴露 set-time/seek-time 后门。

源码审阅未发现 `unsafe`、`TODO`、`FIXME`、`HACK`、`todo!()` 或 `unimplemented!()` 遗留。该搜索只证明受检 Rust 源码没有这些标记，不替代行为或安全审计。

v0.3 schema/fixture/DTO 不留在 active tree；旧版、未来版、旧字段、JSON-LD 和 open tagged union 由结构化负向测试拒绝。依据 ADR 0008，pre-1.0 migration 是 current 0.4 直接替换 current 0.3，不提供 runtime compatibility shim。

收口审阅发现 schema `$id` 指向的 GitHub Pages v0.4 URL 当前返回 HTTP 404，v0.3 历史 URL 和 repository Pages API 同样为 404。JSON Schema 2020-12 明确 URI identifier 不要求可通过网络下载，因此该事实不影响 local schema、loader 或格式合法性，也不阻断 v0.4；但它与 v0.3 closure 曾记录的远端 200/byte-match 状态发生漂移。后续 [#103](https://github.com/illusion-tech/laneflow/issues/103) 已在 Project Backlog 记录，负责选择 identifier-only 或恢复 canonical network publication，并增加对应自动检查。

## 8. 发现项处置与剩余边界

- #97 初次 all-green 预采样相对 no-signals 增加 `20.8%`：审阅定位为全 green 时仍逐车计算 braking horizon/Gate denial；private restrictive snapshot bit 修复后正式 matched 增量降至 `0.267%`。
- restrictive pressure 为 `1.061–1.138 ms/tick`，略高于 soft target 但远低于 hard limit；结果、环境和边界已记录，不以例外伪装为 `<= 1 ms/tick`。
- legacy transition-heavy 相对 v0.3 文档基线 `+25.52%`，触发分析阈值；其他 legacy 同向漂移 `+11–20%`，无项目超过 `30%` 阻断线，当前 v0.4 matched/scaling 未显示孤立或二次回退。
- schema `$id` 网络发布状态漂移由 #103 跟踪；它不成为 runtime loader 的隐式网络依赖。
- Windows 偶发 incremental cache `os error 5` 只影响后续缓存复用；命令退出码为 0 且测试/clippy/doc 完成时不改变验证结论。
- permissive movement、jurisdiction rules、conflict/reservation、adaptive control、Adapter visual 和城市级 partition/parallel 继续属于明确非目标；不得从本 milestone 推断已实现。

## 9. 影响与迁移结论

- **Core API**：v0.4 有意新增 StopLine、MovementGate、SignalController/Group/Phase/Aspect、SignalRegistry、query snapshot 与 signal events；Vehicle Following 的 typed handles、route occurrence、determinism 和 safety contract 保持成立。
- **Data format**：依据 ADR 0008，current format 从 0.3 直接切换到 0.4，新增必填 `signals` 与 closed static signal shape；旧 current 明确拒绝，历史由 Git/closure review 保存。
- **Adapter API**：未新增或冻结 Adapter ABI。未来 Adapter 可消费 query/events 做表现同步，但不得复制 phase authority、permission 或 safety rule。
- **示例/演示**：两个 canonical JSON fixtures 覆盖完整/空 Signals 的 schema、production loader 与 Core 闭环；不包含引擎灯具或 authoring UI。

## 10. 结论

截至本次审阅，未发现阻断 v0.4 Signals 收口的正确性、确定性、数据契约、性能、架构、安全或治理缺口。#18 的五个实施 sub-issues 已完成；唯一新发现 #103 已记录为不影响 JSON Schema 合法性和 runtime 的独立 Backlog 任务。
