**文档状态**: Active

## 1. 结论边界

#106 交付的是无 Parking public types 的 Core 私有 substrate，不是停车功能本身。当前实现证明：

- route distance 对 finite edge 累计溢出安全，并由独立整数 oracle 覆盖；
- vehicle/route resolver removal、stable update order、route reference 与 deterministic compaction 不再对每条 lifecycle command 扫描全部 V；
- command-spatial 查询只访问局部 physical-edge buckets，以 committed `VehicleState` 为 progress authority，并通过 route occurrence 做最终过滤；
- warm lifecycle allocation、10k→100k retained memory 与 operation count 满足 `parking-system.md` 的 #106 门禁；
- fixed-100 command 从 10k 到 100k vehicles 的最终倍率为 `1.576x–1.646x`，低于 `2x` 目标；world 与 command count 同时放大 10 倍为 `14.686x`，低于 `20x` 目标；
- 10k Core step 全场景远低于 `4 ms/tick` 硬上限。route-heavy transition 场景触发 `>5%` profile，但经 hot-path 调整后保持 `<10%` 默认阻断线。

本文不证明 ParkingArea/ParkingSpace、reservation/binding、Parked、ParkingStop、leave/commit、schema 0.5、loader 或 Adapter 已实现，也不构成 100k realtime SLA 或渐近复杂度证明。

## 2. 实现与正确性证据

| 要求 | 实现/自动化证据 |
| --- | --- |
| overflow-safe occurrence distance | `src/route_distance.rs` segmented index；256-case 独立整数 oracle；huge/tiny boundary 与 total overflow 定向测试 |
| stable lifecycle order | 尾部追加、reverse position、tombstone；64-case、每 case 1–128 ops 的 Vec reference model |
| deterministic compaction | 10k high-water test：4,999 tombstones；下一成功 lifecycle command 压缩到 0，live order 保持 |
| exact route reference | exact live count + stable-position lazy heap；slot reuse、stale heap、route-in-use 首个 vehicle 与 reference model 对齐 |
| resolver removal | route/vehicle `IndexMap::swap_remove`；active/stale handle 与 generation tests |
| command-spatial completeness | 256-case local-index vs full-scan oracle；same/adjacent/repeated occurrence、双向 overlap 与 direct followers |
| physical membership sync | 只在 spawn/despawn、completion 或真实 physical-edge transition 同步；同 edge progress 读取 authority state |
| failure atomicity | random lifecycle model、overlap/route-in-use、step failure/retry 与私有 index consistency audit |
| local complexity | 128 与 10k far-background 的 operation counters 完全相同 |

Operation counter 结果：

| Query | edge ranges | occupants visited | 128 vs 10k |
| --- | ---: | ---: | --- |
| overlap | 2 | 2 | identical |
| direct followers | 1 | 1 | identical |

这些 counter 证明选定场景的查询工作只随局部 `k` 变化，不单独构成任意 topology 的复杂度证明；full-scan oracle 负责结果完整性。

## 3. Allocation、retained memory 与依赖审计

`tests/lifecycle_allocations.rs` 使用 dev-only `stats_alloc = 0.1.10` 的 scoped region；测试单线程运行，先 warm capacity，再测 operation：

| Operation | Allocation result |
| --- | --- |
| 100 overlap failures，128 vehicles / route 8 | 200 alloc、0 realloc、2,000 bytes |
| 100 overlap failures，10k vehicles / route 512 | 200 alloc、0 realloc、2,000 bytes |
| 100 warm route-in-use failures | 0 alloc / realloc / bytes |
| successful spawn | 1 alloc、0 realloc、9 bytes owned ID |
| warm despawn | 0 alloc / realloc / bytes |
| warm no-event step | 0 alloc / realloc / bytes |

Retained-memory test 统计 Core 可归属 capacity，不使用 RSS：

| Scale | Accounted bytes | Bytes/live vehicle |
| --- | ---: | ---: |
| 10k | 4,869,570 | 486.96 |
| 100k | 39,878,706 | 398.79 |

10k→100k 比率为 `8.1894x`，低于 `12x` 目标和 `15x` 阻断线。该数包含 lifecycle、route occurrence、route reference、command-spatial 与 reusable scratch 的 retained capacity；不表示进程完整 RSS。

依赖审计：

- `stats_alloc 0.1.10` 只在 `laneflow-core` dev-dependencies，normal graph 中为 0；package 自身没有 dependencies；
- crates.io lock checksum：`5c0e04424e733e69714ca1bbb9204c1a57f09f5493439520f9f68c132ad25eec`；license metadata 为 MIT；
- production source 未增加 unsafe；第三方 crate 的 unsafe 仅用于 `GlobalAlloc` forwarding/instrumentation；
- `cargo-deny 0.20.2`：advisories、bans、licenses、sources 全部通过；官方 Windows archive SHA-256 为 `975a22143262fd27476d19ee00c7af67978426e40e1dee94eed6bbade1cf87dc`。

## 5. Command 性能结果

### 5.1 Fixed-100 isolation

最终 SHA `602db95` 的一次复核：

| Workload | 10k | 100k | 100k / 10k |
| --- | ---: | ---: | ---: |
| safe spawn/despawn | 33.227 µs | 54.683 µs | 1.646x |
| overlap failure | 32.124 µs | 52.526 µs | 1.635x |
| route-in-use failure | 0.908 µs | 1.431 µs | 1.576x |
| mixed churn | 27.629 µs | 44.248 µs | 1.601x |

正式三轮 candidate isolation ratio 的中位数为 safe `1.645x`、overlap `1.599x`、route-in-use `1.420x`、mixed `1.584x`。只有 safe round 2 为 `2.048x`，触发单轮 profile，但远低于 `4x` 阻断线；三轮判定全部达到 `<=2x` 目标。

同一 harness 上，pre-change baseline 的 fixed-100 10k 中位数约为 `2.82–5.45 ms`（route-in-use 约 `0.39 ms`），且 100k/10k 约为 `10.1x–15.5x`。候选把 validation 候选集从全 V 扫描收缩为局部 `k`；这里的数量级改善不改变 public error ordering，因为最终仍按 stable update order 选择错误对象。

## 6. Core step 回归与 hot-path 调整

完整三轮矩阵在 `0b6f380` 上运行；表中 delta 先按 round 做 candidate/base，再取三个 delta 的中位数：

| Scenario | Candidate median ms/tick | Paired delta median |
| --- | ---: | ---: |
| free-flow | 0.726 | +3.14% |
| dense-platoon | 1.109 | +2.07% |
| stop-and-go | 1.121 | +4.64% |
| projection-heavy | 1.086 | -8.25% |
| transition-heavy isolated self-loop | 1.538 | +7.73% |
| no-Signals | 0.970 | +6.29% |
| all-none | 0.989 | +4.86% |
| all-green | 0.986 | +4.69% |
| all-red | 1.160 | +6.04% |
| stop-release | 1.077 | +6.46% |
| mixed-offsets | 1.123 | +4.37% |

所有场景远低于 `4 ms/tick` 硬上限。`>5%` 项进入 profile，未出现 `>10%` 默认阻断项。

首次完整矩阵的 transition-heavy delta 中位数为 `+18.71%`，触发阻断。提交级二分把首次回归定位到 overflow-safe route-distance 数据直接嵌入 `RouteSlot`：三个 `Vec` 扩大 10k 独立 route 的热工作集。修复分两层：

1. distance/reference 派生索引迁移到并行 SoA，恢复紧凑 `RouteSlot`；
2. route-end horizon 先检查当前/下一 edge，常见不可达目标不访问完整 segmented index。

移除 `spatial_changes` 分支和更换 stable-order iterator 的两个实验探针均无改善，因此没有保留无依据的复杂化。最终 SoA source 在提交为 `602db95` 前以 dirty worktree 做定向复测；随后完整 tests/clippy 与提交后 fixed-100 证明 committed tree 一致。结果为：

| Probe | Base | Candidate | Delta |
| --- | ---: | ---: | ---: |
| no-Signals 10k x 60 | 56.713 ms | 57.966 ms | +2.21% |
| transition-heavy 10k x 60 | 88.580 ms | 96.015 ms | +8.39% |

no-Signals 回到 `<=5%` 目标；transition-heavy 是每车每 tick 都发生 self-loop occurrence transition 的非 common pressure 场景，仍在 profile 区但低于 `10%` 阻断线和 `4 ms/tick` 硬上限。保留该残余作为后续同机回归基线，不以跨运行平均值掩盖。

## 8. 剩余边界

- Command index 是 non-authoritative cache；未来 Parking leave 必须继续用 committed state 与 route-aware final validation。
- 两个规模点和 operation counters 用于发现全表/近二次信号，不是数学复杂度证明。
- 100k 是 scaling observation，不是实时服务等级。
- Exact private containers、chunk size、compaction threshold 与 allocation instrumentation 仍可在不改变 public contract 的前提下演进，但必须重跑本协议。
- #108 只能在 #106/#107 G4 后使用本 substrate；不得把 #106 的私有结构公开成 Parking API。
