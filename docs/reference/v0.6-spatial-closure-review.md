## 3. 权威分层与当前生产事实

### 3.1 Core 与 Spatial 权威

- Core 的 `EdgeLength`、`EdgeProgress` 和车辆纵向状态保持 current-f64；Traffic lane graph 是拓扑、车辆归属和交通路径长度权威。
- Spatial 在有界 canonical frame 内使用 LaneFlow-owned `f32` 点、向量、basis 和 pose；Spatial 折线弧长是渲染几何权威，不反写 Core 长度或车辆状态。
- 采样使用 `progress / core_length * spatial_arc_length`。零进度和 Core 端点分别精确映射首点和末点；内部顶点采用右连续分段，末点 tangent 使用最后一条有效入射线段。
- 长度绑定允许量为 `max(0.01 m, core_length * 1e-6) + 0`。末项是 current-f64 的 Core 量化余量；它不会暗中复制 Core crate-private 边界容差。
- 相邻 Traffic edge 的 canonical 端点距离不得大于 `0.005 m`。首版没有隐式吸附、修补、插值桥接或拓扑重写。

### 3.2 Frame、placement 与 Adapter 边界

canonical frame 固定为右手系、Y-up、米制；每个 `SpatialRegistry` 只绑定一个非空 `frameId`。宿主世界坐标、tile、浮动原点和 transform 不进入 Core/Spatial 权威。Adapter 只消费 LaneFlow-owned canonical pose，并用 batch-level `FramePlacementToken` 识别宿主放置版本；旧 token 的批次不能提交。

生产公共 API 未暴露 Bevy、Glam、Euclid、Nalgebra、Unity、Unreal、Godot 或宿主 `Transform` 类型。`laneflow-spatial` 的 normal dependency 只有 `laneflow-core`；Euclid/Glam 只存在于历史 research prototype，未进入生产依赖图。

## 4. 数据制品、schema 与加载原子性

当前有效数据族彼此独立：Traffic Data 为 v0.5，Spatial Package 与 Scenario Manifest 均为 v0.1。Spatial 点使用固定 `[x, y, z]` triple，JSON number 先以 `f64` wire 值解析，检查 finite 和闭区间 `[-16384, 16384] m` 后才规范化为 canonical `f32`。

Scenario loader 的顺序和失败边界已经固定：

1. 严格解析 manifest 版本与 closed shape；
2. 校验 Traffic/Spatial 不同的非空 opaque `artifactRef`、固定 media type、原始 byte size 和 lowercase SHA-256；
3. 先加载 Traffic v0.5，再用其 lane graph 解析 Spatial；
4. 拒绝未知、重复或缺失 `trafficEdgeId`，并按 Traffic graph 稳定顺序生成 dense Spatial 结果；
5. 只有全部步骤成功才返回 `LoadedScenario`，任何错误都不暴露部分结果。

两个 v0.1 schema 均使用 `additionalProperties: false`，Spatial point 同时固定三项、最少三项和最多三项。`schemas/publication.json` 将两个 schema 的 immutable source revision 固定为 `0d7daf6`。主线 `a2054eb` 的 schema publication run #29741374563 已完成构建、Pages 部署，以及 canonical URL 的 HTTPS、media type 和 byte-for-byte 验证；定时 publication monitor 最近一次运行 #29721717002 也通过。

## 5. 几何、批量位姿与失败语义

`SpatialRegistry` 先在暂存映射中验证所有折线、长度、basis、edge coverage 和 5 mm join，再按 lane graph 顺序一次性构造不可变 dense entries。构造失败不发布部分 registry。

Lane/Parking 批量提取先检查 frame/placement identity，使用调用方可复用的 committed/scratch 双缓冲。任一 record 失败都会清空 scratch 并保留上一批 committed output；全部成功才交换 buffers 和更新 token。Parking 的 Core `f64` lateral/heading 只作为受检输入，派生点和方向必须重新满足 canonical `f32` 不变量。

首版明确不包含 CRS、样条、道路倾斜、地形贴合、空间索引、宿主 Transform 同步或 Bevy Adapter；这些边界不是收口缺口。

## 6. 正确性、资源与性能裁决

机器可读证据 [`v0.6-spatial-performance-evidence.json`](v0.6-spatial-performance-evidence.json) 严格解析为 `decision = "pass"`。它冻结在 PR #166 的源提交 `a9dd0df`；该提交仍由 PR 提交链保留，其 Git tree `cfebde0` 与 rebase 后主线等价提交 `67b20a5` 完全相同，因此性能输入可追溯到当前生产实现。

| 规模    | production `f32` p95 | 同形 `f64` p95 | 配对吞吐比 `f32/f64` | 双缓冲 retained bytes 降幅 |
| ------- | -------------------: | -------------: | -------------------: | -------------------------: |
| 10,000  |             0.464 ms |       0.435 ms |             1.047045 |                        45% |
| 100,000 |             5.189 ms |       5.041 ms |             1.037867 |                        45% |

- 两个规模分别低于 `2 ms` / `20 ms` 固定机 p95 上限；10k 到 100k 扩展为 `11.184738x`，低于 `12x` 上限。
- 配对吞吐回退分别为 `4.7045%` / `3.7867%`，均不超过冻结的 `5%` 上限；Criterion 独立诊断中 production `f32` 反而快约 `2.77%` / `2.56%`，两类统计不互相替代。
- 调用方预留稳定容量后，10k/100k 均为 0 allocation、0 reallocation；retained bytes 从同形 `f64` 的 160/1600 万降至 88/880 万。
- 512 个 production `f32` 对离线同构 `f64` oracle case 全部通过：位置误差不超过 `0.01 m`，tangent/up 角误差不超过 `0.5°`。

固定机 wall-clock 轮次不在收口中重跑：关闭审阅核对冻结环境、协议、原始 rounds、来源提交和主线等价 tree；共享验证继续覆盖正确性、确定性、零分配、benchmark compilation 和依赖策略。单机结果不是跨平台 SLA。

## 9. 影响与剩余边界

- **Core API**：无变化；交通长度和进度保持 current-f64。
- **Data format**：无变化；Traffic v0.5、Spatial v0.1 与 Scenario Manifest v0.1 保持当前有效版本。
- **Spatial API**：无变化；继续采用有界 canonical `f32` 和 LaneFlow-owned 类型。
- **Adapter API**：无变化；宿主只消费 canonical pose 并管理 placement token。
- **依赖与许可证**：无变化；没有第三方生产几何 crate。
- **已知限制**：固定机基准不是跨平台 SLA；canonical frame 不是 CRS；首版折线、Y-up 和无 road banking 是明确范围；Core/Data 的 no-go 不改变 Spatial `f32` 裁决。

## 10. 结论

截至本次审阅，没有发现阻断 Spatial 切片收口的正确性、确定性、数据契约、性能、资源、架构、安全、依赖、发布或治理缺口。生产边界一致地保持“Core current-f64 交通权威 + Spatial 有界 canonical `f32` 渲染几何权威”，并以显式比例映射、严格容差和原子失败语义连接两者。
