# v0.5 Parking Runtime 验证基线

**文档状态**: Review
**最后更新**: 2026-07-17
**适用范围**: #108 Parking runtime authority、snapshot、Parked lifecycle、六条同步 command 与 transitional guard
**设计依据**: `../design/parking-system.md`、ADR 0010

## 1. 结论与边界

#108 已实现并验证：

- Core-owned dense forward/reverse Parking authority，以及 global/area 增量 counts；
- 借用 committed world、不会复制全量状态的 `ParkingSnapshot`；
- reserve、cancel、commit、leave、reserved-route rebind、parked spawn 六条同步 command；
- `VehicleStatus::Parked`、lane/parking position authority transfer、Reserved/Occupied despawn release 与 route reference；
- local overlap/direct-follower leave validation、固定 error priority、失败/no-op 原子性；
- committed `reserved_count > 0` 时的窄 `ParkingVehicleCapabilityUnavailable` guard。

本文作为 #108 历史切片不单独证明 moving Reserved step、ParkingStop、step-side arrival、route-completion release 或 Parking events；这些已由 #109 交付，并由 #110 的 [`v0.5-parking-validation.md`](v0.5-parking-validation.md) 完成 milestone 端到端组合验证。Adapter API 仍不属于 v0.5。

## 3. Allocation 与规模证据

`parking_runtime_allocations.rs` 使用 `stats_alloc`；由于计数器是进程全局状态，四项测试标记为 explicit ignored gate，并由 CI 使用 `--ignored --test-threads=1` 串行执行，避免 libtest worker/harness 分配污染 measurement region。以下 warm operation 均为 0 allocations、0 reallocations、0 allocated/reallocated bytes：

- borrowed snapshot 的 count/single/vehicle/ordered iterator traversal；
- reserve、cancel、commit；
- occupied-only steady step。

规模 smoke：

```text
cargo +1.96.0 test -p laneflow-core --test parking_runtime_scale --locked
cargo +1.96.0 test -p laneflow-core --test parking_runtime_scale hundred_thousand_vehicle_parking_runtime_smoke_preserves_counts --locked -- --ignored --exact
cargo +1.96.0 test -p laneflow-core --lib world::tests::lifecycle_retained_memory_10k_to_100k_is_linear --locked -- --ignored --exact --nocapture
```

10k 常规 smoke 通过；100k 显式 Parking smoke 用时 0.69 s。Retained-memory accounting 已包含 runtime space state、vehicle reverse binding 与 area counts：

|       V | accounted bytes | bytes/live vehicle |
| ------: | --------------: | -----------------: |
|  10,000 |       5,525,266 |             552.53 |
| 100,000 |      45,121,922 |             451.22 |

10k→100k ratio 为 `8.1665x`，低于 `<=12x` 目标；未出现 V×S 或 per-vehicle route/space 副本。

## 6. Parking workload scaling

Fixed command workload 使用 100 个 vehicle/space pair，每次 batch 执行 100 reserve + 100 cancel。场景先 clone 一次并 warm；计时区只反复执行精确恢复 authority 的 roundtrip，不计 world clone/destructor。早期按值 batched 测量会把大 world 析构/cache pollution 混入，已判定为 benchmark defect并废弃。

| workload                 |    10k median (95% CI) |      100k median (95% CI) |  ratio |  gate |
| ------------------------ | ---------------------: | ------------------------: | -----: | ----: |
| 200 reserve/cancel calls | 1.510 us (1.497-1.547) |    1.519 us (1.494-1.537) | 1.006x |  <=2x |
| occupied-only 60 steps   | 5.773 ms (5.748-5.923) | 80.898 ms (76.391-98.748) | 14.01x | <=20x |

100k command run报告 1 个 high-severe outlier（10 samples 的 10%）；其 median ratio 仍接近 1。Occupied-only 100k confidence interval 较宽，属于本机 power/noise 风险，但未接近 20x 阻断线；#110 已在最终 activation workload 上完成三轮 10k/100k matched 重测，见 [`v0.5-parking-validation.md`](v0.5-parking-validation.md)。

## 7. 历史边界处置与 Remaining risks

- #108 的 Reserved step guard 已由 #109 在完整 activation 同一切片解除；合法 current production world 可执行 moving Reserved step。
- #110 profile 确认移除当前最快 Active vehicle 会让 stale max-speed 扩大 local reverse horizon；current 实现已升级为 command-phase on-demand exact max heap，把 fixed batch 从 `O(B x V)` 收敛到 `O(V + B log V)`，且保持正常 tick scalar fast path。证据见 [`v0.5-parking-validation.md`](v0.5-parking-validation.md)。
- `ParkingReleaseReason::RouteCompleted` 和 Parking step events 已由 #109 交付；#108 本身的历史测试仍不产生这些 observable。
- 100k 仍只是扩展性观察，不是 realtime SLA；Windows Balanced plan 与 Insider build 会增加 timing noise。
