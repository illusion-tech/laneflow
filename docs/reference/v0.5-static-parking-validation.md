# v0.5 Static Parking 与 current data 验证基线

**文档状态**: Active
**最后更新**: 2026-07-17
**适用范围**: #107 static Parking registry、current 0.5 data、normalization/rebind 与 step 性能回归
**过程证据**: [GitHub Issue #107](https://github.com/illusion-tech/laneflow/issues/107)
**设计依据**: `../design/parking-system.md`、ADR 0008、ADR 0010

## 1. 结论边界

#107 已形成可执行的 current 0.5 static/data 基线：

- Core 提供 immutable `ParkingRegistry`、area/space opaque handles、stable resolvers/member order、entry/exit anchors 与 edge-relative geometry；
- `InitialTrafficData` 按自身 LaneGraph 重新解析 Parking handles，`CoreWorld::parking()` 只读暴露同一 registry；
- production loader 只接受 0.5，严格解析必填 Parking shape，并区分 omitted `areaId` 与 explicit `null`；
- active tree 只保留 0.5 schema 与两个 canonical fixtures；v0.4 直接返回 version error；
- 10k all-vacant spaces 不改变 10k vehicle step 结果，不增加 warm-step allocation，也不形成 per-step `S` 工作。

本文不证明 reservation/occupancy、commands、ParkingStop、`VehicleStatus::Parked`、arrival/leave/release events 或 Adapter API；这些由 #108-#110 交付。GitHub Issue/PR 继续负责 Gate、review、final-head commit、CI 与合并时序。

## 3. Matched workload 与门禁

两侧均使用 10,000 Active vehicles、400 matched routes、16 ms fixed tick、连续 60 ticks；all-vacant 侧额外持有 10,000 standalone ParkingSpaces。Space anchors 均预解析到现有 entry edge，runtime 没有 reservation/binding。

门禁：

- #106 merge head -> candidate empty registry：目标 `<=5%`，`>10%` 阻断；
- candidate empty registry -> 10k all-vacant：目标 `<=5%`，`>10%` 阻断；
- 10k common 目标 `<=1 ms/tick`，hard limit `<=4 ms/tick`；
- warm no-event step 必须 0 allocation；发现 per-step/per-vehicle full-S pass 直接阻断。

Criterion 每轮 20 samples、1 s warm-up、5 s measurement，读取 `estimates.json` 的 `median.point_estimate`，先计算同轮 paired delta，再取三轮 delta 中位数。Console interval 不能代替 median；baseline names 与完整 artifacts 保存在各 worktree 的 `target/criterion`。

## 5. 三轮结果

### 5.1 Empty registry -> 10k all-vacant

单位均为 60 ticks 的 median；括号内为 median confidence interval。

| Round |                     Empty |            10k all-vacant | paired delta | outliers empty/vacant |
| ----- | ------------------------: | ------------------------: | -----------: | --------------------: |
| 1     | 65.955 ms (65.622–66.734) | 66.068 ms (65.484–66.828) |      +0.171% |                 3 / 1 |
| 2     | 56.598 ms (56.312–57.402) | 58.486 ms (57.938–60.486) |      +3.336% |                 0 / 0 |
| 3     | 57.390 ms (57.161–57.716) | 59.328 ms (58.518–59.967) |      +3.378% |                 2 / 0 |

三轮 paired delta 中位数为 **+3.336%**，满足 `<=5%` 目标。两侧三轮 median 分别为 57.390 ms / 59.328 ms，即约 0.956 ms / 0.989 ms per tick，均满足 common target。

### 5.2 #106 merge head -> candidate empty registry

| Round |                      Base |                 Candidate | paired delta |
| ----- | ------------------------: | ------------------------: | -----------: |
| 1     | 59.132 ms (58.850–59.690) | 59.139 ms (58.073–59.800) |      +0.011% |
| 2     | 68.599 ms (67.565–70.271) | 62.356 ms (61.307–63.266) |      -9.100% |
| 3     | 60.715 ms (59.905–62.230) | 60.428 ms (59.901–60.891) |      -0.474% |

三轮 paired delta 中位数为 **-0.474%**，没有结构性回退。Round 2 显示明显同机环境漂移，但方向不是 candidate regression；因此判断以交错 paired delta、同轮 static comparison、0-allocation test 和 hot-path code audit 的组合证据为准。

## 6. Allocation 与 complexity audit

`parking_static_scale.rs` 在两个 10k worlds 完成 warm-up 后测量单 step：empty/all-vacant 均为 0 allocations、0 reallocations、0 allocated/reallocated bytes。60 ticks 的 `StepResult` 和全部 VehicleState 逐 tick 相等。

Static normalization 是 `O(A + S + lookup)`，retained state 为 area/space definitions、dense maps、resolved anchors 与 ordered reverse members。`CoreWorld::step` 不读取 `parking` 字段；不存在 vehicle x spaces、route x spaces、full-S candidate copy 或 per-tick external-ID lookup。
