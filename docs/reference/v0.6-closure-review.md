## 1. 目的与结论

本文汇总已经独立完成的数值切片与 Spatial 切片，判断 v0.6 是否满足整体完成定义。它不重新解释各切片的原始机器证据，也不新增运行时、数据格式或 Adapter 决策。

截至本次审阅，v0.6 的设计、生产实现、数据制品、正确性、性能、安全、发布和治理链均已闭环。最终生产边界不是“全局统一使用一种浮点类型”，而是 Core/Data 保持 current-f64 交通权威，Spatial 使用有界 canonical `f32` 几何与位姿权威，并通过受检比例映射和明确的 Adapter placement 边界连接两者。

## 3. 最终生产契约

| 领域          | v0.6 完成后的唯一当前事实                                                                                               |
| ------------- | ----------------------------------------------------------------------------------------------------------------------- |
| Core 数值权威 | `EdgeLength`、`EdgeProgress`、速度、加速度及交通高频状态保持 current-f64；九个领域判定 owner 保持显式分离               |
| Traffic Data  | v0.5 继续是唯一有效 Traffic 版本；没有生产 v0.6 loader、schema、精度开关或双权威字段                                    |
| Spatial frame | 米制、右手、Y-up、稳定 `frameId`、每轴闭区间 `[-16384, 16384] m` 的 canonical `f32`                                     |
| 长度与采样    | Core 长度拥有交通进度；Spatial 弧长拥有几何参数化；使用比例映射，首尾端点精确，内部顶点右连续                           |
| 容差          | 长度绑定 `max(0.01 m, core_length * 1e-6)`；相邻 edge 端点最多 `0.005 m`；没有隐式吸附或全局通用 epsilon                |
| 数据制品      | Spatial Package v0.1 与 Scenario Manifest v0.1 以不同 opaque ref、原始 byte size、固定 media type 和 SHA-256 配对       |
| Adapter 边界  | Adapter 消费稳定批量 canonical pose 与 `FramePlacementToken`；宿主世界坐标、Transform、tile/浮动原点不进入 Core/Spatial |
| 生产几何依赖  | `laneflow-spatial` 的 normal dependency 只有 `laneflow-core`；公共 API 不泄漏第三方或宿主引擎类型                       |

ADR 0014 继续保存未来 Core/Data target-f32 的目标、迁移和回退门槛，但不构成当前生产入口。ADR 0015 对 Spatial 的有界 canonical `f32` 决策独立成立，不因 Core/Data no-go 回退。

## 4. 正确性、资源与性能裁决

数值完整生产候选通过正确性与内存护栏，但 14 项稳态矩阵的等权几何平均耗时只降低 `4.257138%`，没有达到预先冻结的 `5%` 实质收益门槛，因此完整回退 current-f64。该 no-go 是本轮产品门槛裁决，不表示 target-f32 永久禁止。

Spatial production `f32` 对 512 个同构离线 `f64` oracle case 全部通过，位置误差不超过 `0.01 m`，tangent/up 误差不超过 `0.5°`。冻结固定机证据中：

- 10k/100k p95 为 `0.464 ms` / `5.189 ms`，均低于 `2 ms` / `20 ms` Gate；
- 10k 到 100k 扩展为 `11.184738x`，低于 `12x` Gate；
- production `f32` 对同形 `f64` 的配对吞吐回退为 `4.7045%` / `3.7867%`，不超过 `5%`；
- 预留稳定容量后 10k/100k 均为零 allocation/reallocation，双缓冲 retained bytes 降低 `45%`。

这些固定机结果不是跨平台 SLA。共享 CI 长期阻断正确性、确定性、零分配、benchmark compilation、schema 与依赖策略。

## 5. 数据、安全与发布状态

Spatial/Scenario loader 严格执行 manifest/descriptor/raw-byte identity、Traffic-first、完整唯一 edge coverage、`f64` wire 检查后受检 `f32` 规范化，以及全场景失败原子性。两个 v0.1 schema 均为 closed shape，Spatial point 固定为 `[x, y, z]` triple；canonical URL 发布采用 immutable-permanent retention。

完成输入 `main@d861bd7` 的 CI、Dependency policy 与 CodeQL 全部成功；2026-07-21 审计的 open CodeQL、secret-scanning 与 Dependabot alerts 均为 0，secret scanning、push protection 与 Dependabot security updates 已启用。最新 canonical schema publication 与定时 monitor 均成功。本次总收口不修改 schema、catalog、依赖、扫描设置或权限。

## 7. v0.7 进入边界

#120 G4 后，#121 的“v0.6 已完成”前置成立，`docs/design/adapter-api.md` 也已接受。但 #121 仍须完成 Bevy 版本、最小 feature graph、MSRV、许可证/RustSec/cargo-deny 审计，并拆分 Adapter、example、validation 与 closure Issue，才能进入 Ready/G1/G2。

因此 v0.6 完成只解除前置阻塞，不等于已选择 Bevy 版本、不授权实现，也不改变 #72 的 active-agent partition 与 simulation-fidelity 研究边界。

## 8. 结论

没有发现阻断 v0.6 整体收口的剩余缺口。数值 no-go 与 Spatial `f32` pass 来自不同产品边界和独立证据，二者共同形成可供首个 Adapter 消费的稳定基础。#120 Delivery PR 通过 required checks、G3 和 main 复验后，可以执行 #120 G4、关闭 Milestone #6，并在 #121 回写“v0.6 前置已解除但仍保持 Backlog”。
