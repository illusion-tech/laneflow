## 4. Authority、原子性与确定性

最终权威边界与 ADR 0016 一致：

- Core 拥有 vehicle state、identity、route、occupancy、speed limit、Signals 和原子 replace invariant，但不拥有目标人口、seed 或入口选择。
- `laneflow-scenario` 的可选 reference policy 拥有 v0.8 的目标人口、stable catalog normalization、SplitMix64 state、portal/lane 决策和 blocked retry；未来宿主可以替换它。
- Adapter 拥有 `VehicleHandle <-> Entity` 部分双射和 `Lifecycle -> Step -> Observe` schedule transaction；它不复制交通规则。
- Spatial 拥有 canonical centerline/frame/pose，Presentation 只拥有 proxy、mesh、Transform、灯具与 HUD。

Core replace 和 Adapter binding 采用预检后提交。`Blocked` 与 fatal validation 都保持 committed authority 不变；成功后 mapping 在同一公开事务中从 old handle 轮换到 new handle。一个 outer frame 运行多个 catch-up step 时，每个 fixed step 都重复 lifecycle、Core step 和 ordered completion consumption，因此 outer-frame 分块不改变 Core/population replay。

同 seed、catalog、fixed-step input sequence 与同一实现版本得到相同 initial/recycle 决策。初始 Fisher–Yates、portal-first/lane-second draw order、simultaneous completion 顺序和 blocked retry 不消耗新随机数均由 golden tests 固定；该承诺不是跨版本或跨平台 bit-level 兼容承诺。

## 8. 兼容性与非目标

- #195 本身的影响为 `core-api=none; data-format=none; adapter-api=none`；它只归档 #185–#189/#203 已交付的 pre-1.0 变化。
- v0.8 当前 Traffic contract 为 v0.7；SpatialPackage/ScenarioManifest 继续为 v0.1。旧 published schema 保持历史可取回，但 production loader 只接受 current version。
- Bevy Reference Adapter 继续支持 Bevy `0.19.x` 和 Rust `1.96.0`；`signalized_corridor` 是 opt-in `native-example` target，不扩大默认 dependency graph。
- Windows native smoke 与共享 CI 编译不构成 macOS、WASM、mobile、Wayland 或第二 Engine Adapter 支持承诺。
- 左转、右转、掉头、lane change、permissive/unprotected turn、adaptive signals、通用 pathfinding 和城市级扩展不属于 v0.8；受保护转向由 v0.9 #194/#196/#190–#192 跟踪。
- 保存/恢复完整 population controller state、跨版本 replay 稳定性、跨机器性能 SLA 与通用人口模型保持后续范围。
